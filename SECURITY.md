# Security Policy

## Supported Versions

This website is actively supported only on the current live deployment and the latest source version used to publish it.

| Version / Deployment | Supported |
| -------------------- | --------- |
| Current live site    | ✅ |
| Latest main branch / latest publish-ready build | ✅ |
| Older deployed builds | ❌ |
| Archived, experimental, or local copies | ❌ |

## Reporting a Vulnerability

If you believe you have found a security vulnerability affecting this website, please report it privately.

### How to report
Please send:
- a clear description of the issue
- steps to reproduce it
- the affected page or feature
- screenshots or proof of concept, if helpful
- the potential impact
- any suggested mitigation, if known

### Where to report
Please report vulnerabilities through one of these private channels:
- **Email:** `security@yourdomain.com`
- **GitHub Security Advisory:** if this repository is public and advisories are enabled

Please **do not** report security vulnerabilities through public GitHub issues, public discussions, or social media.

### What to expect
- You should receive an acknowledgement within **3–5 business days**
- We will review the report, assess severity, and determine next steps
- If the report is accepted, we will work on a fix and may contact you for clarification
- If the report is declined, we will explain why when possible

### Disclosure
We ask that you keep vulnerability details private until a fix has been released or mitigation is in place.

We appreciate responsible disclosure and efforts to help keep this website and its users safe.
